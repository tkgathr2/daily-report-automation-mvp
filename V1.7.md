# V1.7（完了条件C：手動動作確認チェックリスト）

## 目的
簡単日報くん（WEBアプリ版）を、完了条件C（取得→表示→編集→コピー→Slack投稿（Webhook設定含む）まで動作確認）まで完了させる。

## 使い方（最短）
1) `V1.6.md` を開く  
2) Rawリンクから `CLAUDE_CODE_Ver12_DEPLOY_VERIFY_INSTRUCTION.md` を開いて、全文を参照しながら進める  

## 注意（必ず守る）
- Slack Webhook URL はチャットに貼らない（伏せる）
- Slack投稿先チャンネルは Incoming Webhook 作成時に固定（画面からチャンネル指定しない）
- Script Properties のキー名は `SLACK_WEBHOOK_URL`

## 手動作業（GAS/Slack）
### 1) GAS新規プロジェクト
- `https://script.google.com/` で新規プロジェクト作成（名前は任意）

### 2) ファイル貼り付け
- GASの `Code.gs` を `src/Code.gs` の内容で全置換
- HTMLファイルを追加して名前を `Index` にする
- `Index.html` を `src/Index.html` の内容で全置換

### 3) Slack Incoming Webhook
- Slackで Incoming Webhook を作成（投稿先チャンネルを固定で選ぶ）
- Webhook URL を取得（共有しない）

### 4) Script Properties
- GAS「プロジェクトの設定」→「スクリプト プロパティ」
- `SLACK_WEBHOOK_URL` = Webhook URL を設定

### 5) Webアプリとしてデプロイ
- 「デプロイ」→「新しいデプロイ」→「ウェブアプリ」
- 実行：自分 / アクセス：まずは自分のみ
- デプロイURLを取得

## 動作確認（完了条件C）
デプロイURLをローカルPCブラウザで開き、順番に確認する：
1) 画面表示：OK  
2) 取得：「今日の予定を取得」→ 0件でもエラーにならない：OK  
3) 編集：テキストエリア編集できる：OK  
4) コピー：コピー→貼り付け確認できる：OK  
5) Slack投稿：「Slackに投稿」→ Slackに投稿される：OK  

## 報告テンプレ（この形式で返す）
- 画面表示：OK/NG
- 予定取得：OK/NG（表示例を2〜3行だけ）
- 編集：OK/NG
- コピー：OK/NG（貼り付け確認OK/NG）
- Slack投稿：OK/NG（投稿本文を貼る／Webhook URLは伏せる）
- 失敗時：エラー文をそのまま貼る
