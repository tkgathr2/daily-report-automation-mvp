## SEC-01（シークレット注意）

この工程ではシークレット情報が必要になる可能性があります。自動入力は行いません。SEC-01 で停止します。

例：
- Script Properties の値（Webhook URL等）が表示される画面に入った
- Slack管理画面等でWebhook URLが表示された

検知したら即停止し、ユーザーに手動対応を依頼すること（値のコピー/貼り付けはしない）。

---

## 手順（V0.3）

### 1) 画面表示の確認

1. 上のURLを開く
2. 画面が白画面にならず表示されること
3. ヘッダーが `【YYYY年MM月DD日　氏名　日報】` 形式で表示されること
4. 4セクションが表示されること
   - 今日やったこと
   - わかった事・問題・共有事項
   - 売上・利益・経費削減に関わるポイント
   - 次すること

### 2) カレンダー自動反映＋再取得（当日固定）

1. 初期表示で「今日やったこと」に当日の予定が入っているか確認
2. 「今日の予定を取得」ボタンを押す
3. 「今日やったこと」が更新され、取得成功メッセージが出ることを確認

### 3) 必須バリデーション（空白のみ不可）

1. 4セクションのうち、1つを空欄（または空白だけ）にする
2. 「Slackに投稿」を押す
3. 送信されず、未入力エラーが表示されることを確認

### 4) コピー（テンプレ完全一致）

1. 「コピー」ボタンを押す
2. 別のメモ欄等に貼り付けて、以下が含まれることを確認
   - 4セクション見出しがテンプレ通り
   - 見出し文言が新名称（売上・利益・経費削減…）
   - 箇条書き記号が `⚫︎`（表記ゆれ無し）

### 5) Slack送信（Incoming Webhook）

1. 4セクションすべてに1文字以上入力する（空白のみは不可）
2. 「Slackに投稿」を押す
3. Slack側で投稿が届くことを確認
