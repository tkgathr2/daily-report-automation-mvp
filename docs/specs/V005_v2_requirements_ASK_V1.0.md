## 1. 目的・背景

### 1.1 目的

現行MVP（カレンダー予定→編集→Slack送信）を土台に、
**「作業ログの自動集約」と「人の気づきの回収」** を両立する日報システムに拡張する。

### 1.2 背景・課題

- 日報を「書く」負担が大きい
- 何をやったか思い出すのが面倒
- 書く内容がバラバラになりがち
- 管理側が「今日何が起きたか」を把握しづらい

### 1.3 V2で解決したいこと

- **作業ログはAIが自動で日報化** → 書く負担をなくす
- **人は「気づき・問題」だけを書く** → 考えるポイントだけ残す
- **事実ベースで「今日何をやったか」を把握** → 管理側も楽になる
