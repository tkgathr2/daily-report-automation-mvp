# V005 簡単日報くん V2（構想）要件定義（Ask）たたき台 V1.0

## 進捗
- 現在フェーズ：Ask（要件定義）
- 根拠：現行 `docs/plan.md` とV2構想のスコープ差分が未確定のため、確定質問が先

---

## 1. 目的（いただいた文章からの抜粋）
- 日報を書く負担をなくす
- 事実ベースで「今日何をやったか」を把握する
- スタッフに「考えるポイントだけ」を残す

---

## 2. V2構想の要点（機能の分解）
### 2.1 事実ログ（AIが自動作成）
- 外部ツールの利用履歴（例：Slack／メール／Googleドキュメント／Drive／Dropbox／Notion／Zoom／Google Meet／社内ツール）を集約
- 重要度順に最大10件を自動入力
- ユーザーは削除・軽微な修正のみ（ゼロから考えない）

### 2.2 気づき・問題・共有事項（人が必須入力）
- 必ず1行以上入力
- AIでは作らない

### 2.3 次すること
- 基本は空欄
- 翌日の「今日やった事」へ自動反映（前提）

### 2.4 個人最適化
- ツール別に表示ON/OFF
- よく使うツールほど上に表示（自動学習）

---

## 3. 現行仕様（`docs/plan.md`）との関係で発生している矛盾
`docs/plan.md` は現状、
- 取得元：Googleカレンダー（当日予定）
- 出力：編集UI → Slack送信（Webhook）
をMVPとして定義しています。

一方、V2構想は
- 取得元：複数ツールの利用履歴（作業ログ）
- 出力：重要度順に最大10件（＝要約・順位付けの要件が増える）
- 追加：個人最適化（学習/順位自動化）
を含み、スコープが大きく異なります。

**この矛盾が解消されない限り、追加機能の優先度決定は推測になり得るため停止します。**

---

## 4. 追加機能の優先度決定の基準（確定が必要）
優先度は、下記4軸の合計で決めます（各1〜5、合計が高いほど優先）。
- 利用頻度（毎日使うか）
- 効果（作業時間削減・送信漏れ防止・マネジメント可視化）
- 難易度（実装・運用・権限/認証・保守）
- 依存関係（先に必要な土台があるか）

※各軸のスコアリングには「対象組織で実際に使うツール」「導入制約」が必要です。

## 5. 確認質問（Yes/No または選択肢）※最大10問
### Q1（最重要）：V2構想は現行MVP（カレンダー→Slack）を置き換えますか？
- A：置き換える（V2を新MVPにする）
- B：置き換えない（現行MVPは継続し、V2は将来拡張/別フェーズ）

### Q2：V2で「事実ログ」の一次ソースとして最優先にしたいのはどれですか？（最大2つ）
- A：Slack
- B：Googleカレンダー
- C：Googleドキュメント/Drive
- D：メール
- E：Notion
- F：会議ツール（Zoom/Google Meet）
- G：その他（社内ツール等）

### Q3：日報の送信先はSlackのみで確定ですか？
- Yes：Slackのみ
- No：将来は他（メール等）も必要

### Q4：「重要度順 最大10件」は、誰の観点での重要度ですか？
- A：本人（自分にとって重要）
- B：上長（管理に重要）
- C：AとBを両立（ルールが必要）

### Q5：「気づき・問題・共有事項」1行必須は強制で確定ですか？
- Yes：送信ボタンを押す前に必須
- No：任意（推奨に留める）

### Q6：「次すること」を翌日の事実ログへ自動反映は必要ですか？
- Yes：必要
- No：不要（別枠で残す/任意）

### Q7：個人最適化（ツールON/OFF）は必須ですか？
- Yes：必須（初期から）
- No：後回し（固定表示で開始）

### Q8：自動学習（よく使うツールほど上に表示）は必須ですか？
- Yes：必須
- No：後回し（手動並び替え等で開始）

### Q9：対象ユーザーは単一ユーザー（自分用）ですか？複数ユーザー（社内配布）ですか？
- A：単一ユーザー
- B：複数ユーザー

### Q10：外部ツール連携で「ログイン/OAuth」が増えることは許容できますか？
- Yes：許容できる（運用で回せる）
- No：許容しにくい（初期は増やしたくない）
