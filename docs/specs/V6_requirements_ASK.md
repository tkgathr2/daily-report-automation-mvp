## 1. 目的・背景

### 1.1 目的

V5までで取得した大量の履歴を**AIで自動処理**し、
重要度順に並べ替え、最大10件に絞り込み、
ユーザーは「削除・軽い修正のみ」で済むようにする。

### 1.2 背景

- V5までで取得できる履歴が大量になる可能性がある
- 全てを日報に載せると読みにくい
- AIで重要度を判定し、自動で絞り込むことで品質を維持

## 3. 機能詳細

### 3.1 重要度順の自動並べ替え（B1）

#### 概要
AIを使って履歴の重要度を判定し、重要な順に並べ替える。

#### 重要度判定基準（例）
- 顧客関連の作業 → 高
- 会議参加 → 中〜高
- 社内コミュニケーション → 中
- 定型作業 → 低

#### 使用AI
- OpenAI API（GPT-4o-mini推奨）
- または Google Cloud AI

#### 入力
- 取得した履歴一覧（V5の結果）

#### 出力
- 重要度スコア付きの履歴一覧

---

### 3.2 最大10件に絞り込み（B2）

#### 概要
重要度順に並べ替え後、上位10件のみを表示する。

#### 仕様
- 重要度スコアの高い順に10件を選択
- 同スコアの場合は時間順（新しい順）
- 10件未満の場合は全件表示

---

### 3.3 削除・軽い修正のみで済むUI（B3）

#### 概要
AIで絞り込まれた履歴に対して、ユーザーは削除ボタンまたは軽い修正のみで済むようにする。

#### UI仕様
- 各項目に「削除」ボタン
- 項目をクリックで編集モード
- ドラッグ&ドロップで順序変更（オプション）

## 4. 受け入れ条件

| No | 受け入れ条件 |
|----|--------------|
| 1 | 履歴が重要度順に並べ替えられる |
| 2 | 表示が最大10件に絞られる |
| 3 | 各項目を削除できる |
| 4 | 各項目を編集できる |
| 5 | AI処理のON/OFFが設定できる |
| 6 | V5機能が正常に動作する |

---

## 5. NG例

| No | NG例 | 理由 |
|----|------|------|
| 1 | AI APIエラーでアプリ全体が動かなくなる | 影響範囲が大きい |
| 2 | 処理に30秒以上かかる | UX悪化 |
| 3 | V5機能が動かなくなる | 後方互換性 |

---

## 6. 例外ケース

| No | 例外ケース | 対応方針 |
|----|------------|----------|
| 1 | AI API Key未設定 | AI処理をスキップし、時間順で表示 |
| 2 | AI API エラー | AI処理をスキップし、時間順で表示 |
| 3 | 履歴が10件以下 | 絞り込みせずそのまま表示 |
